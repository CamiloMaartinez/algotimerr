O(n²)

Cuadrática. Típico en algoritmos de ordenamiento simples como Bubble Sort o Selection Sort. El tiempo crece con el cuadrado de la entrada.

n=1000 → 10⁶ ops

O(n log n)

Linealítmica. El estándar de eficiencia para ordenamiento. Algoritmos como Merge Sort, Quick Sort y Heap Sort pertenecen a esta clase.

n=1000 → ~10,000 ops

O(n³)

Cúbica. Aparece en multiplicación de matrices naive, Floyd-Warshall y algunos problemas de programación dinámica. Muy costoso para n grande.

n=1000 → 10⁹ ops

O(2ⁿ)

Exponencial. Problemas NP como la mochila, subset-sum o el viajante. Impracticable para n > 40. Solo viable con entrada muy pequeña.

n=40 → 10¹² ops
